# Week 2: Object-Oriented Programming (OOP) in Python

## Overview

This week focused on understanding Object-Oriented Programming (OOP) concepts in Python. OOP is a programming paradigm that organizes code into reusable objects and classes, making applications more structured, maintainable, and scalable. Since many machine learning frameworks and production systems are built using OOP principles, mastering these concepts is an important step in becoming an effective ML engineer.

## Topics Covered

### 1. Introduction to OOP

* What is Object-Oriented Programming?
* Why Python is considered an Object-Oriented Language
* Real-world examples of objects and classes
* Benefits of using OOP

### 2. Classes and Objects

* Creating classes
* Creating objects from classes
* Understanding attributes and methods
* Accessing object properties and functions

### 3. Instance Attributes and Methods

* Defining attributes inside classes
* Using constructors (`__init__`)
* Working with instance variables
* Calling methods through objects

### 4. Core OOP Concepts

#### Encapsulation

* Bundling data and methods together
* Restricting direct access to data

#### Inheritance

* Reusing functionality from existing classes
* Parent and child classes

#### Polymorphism

* Using the same interface for different implementations

#### Abstraction

* Hiding implementation details and exposing essential functionality

### 5. Advantages of OOP

* Improved code reusability
* Better maintainability
* Easier debugging and testing
* Modular application design
* Enhanced collaboration in large projects
* Ability to create custom data types and structures

## Key Outcomes

By the end of Week 2, I was able to:

* Understand the fundamentals of Object-Oriented Programming.
* Create and use classes and objects in Python.
* Implement attributes, methods, and constructors.
* Explain the four pillars of OOP.
* Design reusable and modular code structures.
* Recognize how OOP principles are applied in machine learning libraries and production software systems.
